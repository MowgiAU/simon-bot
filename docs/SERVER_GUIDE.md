# Fuji Studio Server Guide

Welcome to the Fuji Studio Community Guide. This document explains the main features of the Discord bot and website so your members can quickly understand what the server offers and how to use it.

## 1. What Fuji Studio Does

Fuji Studio is a modular Discord bot built for music producers. It helps your community with:

- Moderation and clean community management
- Member onboarding and verification
- Music production support and FL Studio help
- Collaborative events like beat battles and featured content
- Gamified economy and leveling
- Website dashboards for managing the server and user activity

## 2. How the Discord Bot Helps Members

### 2.1 Getting Started

- Join the server and complete verification if required.
- Use `/help` or ask a moderator if you need the current command list.
- The bot uses slash commands for most actions, so start typing `/` in chat.

### 2.2 Moderation & Safety

These features keep the server healthy:

- `/kick`, `/ban`, `/timeout`, `/warn`, `/purge` — moderation actions for staff
- Word filtering and channel rules — keeps unwanted content out of public channels
- Anti-piracy protection — detects and handles piracy-related posts automatically

### 2.3 Onboarding & Verification

- New members may be placed in an unverified role until they complete verification.
- Verification is usually done by clicking a button and answering a few questions.
- Once verified, members gain access to the main community channels.

### 2.4 Leveling & Economy

Your members can earn rewards for active participation:

- XP for messages, voice chat time, reactions, and streaks
- Level-based role rewards can be configured by staff
- Economy currency can be earned through activity and spent in the server shop
- Tip other members or reward helpful contributors

### 2.5 Help & Support

- `Studio Guide` AI answers FL Studio, music production, mixing, and music theory questions in a dedicated help channel.
- Users can opt out if they don’t want AI responses.
- Staff can pause Studio Guide when human help is preferred.

### 2.6 Ticketing & Support Requests

- Users can open private support tickets in the server.
- Staff manage tickets through private threads and support categories.
- Transcripts can be recorded when tickets are closed.

### 2.7 Music Production & Community Events

- Beat Battles: community contests where producers submit tracks and members vote.
- Featured content: highlight top tracks, community picks, or producer spotlights on the website.
- Fuji FM Radio: community radio with auto-play and live host modes, plus track tipping and likes.

### 2.8 Bot Messaging & Announcements

- Staff can use the bot to send rich embed messages across channels.
- The dashboard includes tools for building announcements and posts.

## 3. How the Website Works

The website extends the bot experience with stronger management tools and public-facing features.

### 3.1 Dashboard Access

- Server staff can log in and use the dashboard to configure the bot and community tools.
- The dashboard organizes features into sections such as moderation, economy, studio guide, beat battles, and more.
- All settings are tied to your server and apply immediately.

### 3.2 User Accounts

- Members can sign up and link their Discord account.
- The website stores account preferences, profile info, and participation data.
- Admins can manage accounts from the dashboard.

### 3.3 Beat Battles & Contests

- Beat battles are managed from the website.
- Producers submit entries through the website and community members vote.
- Results are announced in Discord and on the site.

### 3.4 Music Library & Featured Content

- The public site can display approved tracks, featured artists, and community highlights.
- Staff choose what content appears on the discovery page.

### 3.5 Studio Guide AI Knowledge Base

- The website can house the AI knowledge base for FL Studio and music production questions.
- Staff can manage Studio Guide settings and knowledge entries from the dashboard.

### 3.6 Email & Notifications

- The dashboard can receive and display incoming email messages.
- Optional Discord notifications alert staff when new mail arrives.

### 3.7 Analytics & Logs

- The dashboard tracks server activity, message counts, voice usage, and moderation actions.
- Audit logs help staff review actions taken by the bot and moderators.

## 4. Public Guide for Members

Use this summary as a pinned message or FAQ in your server.

### Important member commands

- `/guide ask <question>` — ask the Studio Guide AI for production help
- `/rank` or `/leaderboard` — check your level and community standing
- `/ticket open` — open a support ticket with staff
- `/radio np` — see the current track on Fuji FM Radio
- `/battle info` — view active beat battles and how to submit

### Where to find help

- Ask questions in the help channel for Studio Guide AI
- Contact staff via ticket if you need personal support
- Use the dashboard only if you have staff/admin access

## 5. Recommended Server Guide Layout

For a Discord public guide, organize the message into:

1. Welcome and purpose
2. How to verify and join
3. Main bot features
4. Website / dashboard benefits
5. How to get help
6. Community rules reminder

## 6. Tips for Staff

- Keep the guide pinned in the most visible channel.
- Update the guide when you add new features.
- Use the website dashboard for settings and event management.
- Encourage members to use `/guide ask` before asking basic production questions.
